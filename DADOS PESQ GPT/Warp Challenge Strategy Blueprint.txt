# VIREON + Warp Challenge Strategy Blueprint

## 1. Leitura e Interpretação do Desafio

**Fonte Oficial**: [https://github.com/warpdotdev/hiring-challenge/blob/main/mission\_challenge.md](https://github.com/warpdotdev/hiring-challenge/blob/main/mission_challenge.md)

### Resumo do Desafio:

Warp forneceu um arquivo `.log` contendo uma lista de "missões espaciais". O objetivo é extrair e apresentar informações relevantes de forma clara e interativa.

**Critérios Chave:**

* Clareza no código
* Usabilidade
* Eficiência
* Criatividade

---

## 2. Arquitetura Proposta (EON + Warp)

### Camadas:

1. **Ingestão e Parsing**

   * Ferramentas: Python, regex, pandas
   * Objetivo: Estruturar o `.log` em um `DataFrame` indexado por data, missão, status, etc.

2. **Sistema de Regras Inteligentes (VIREON-core)**

   * Validação e enriquecimento dos dados via regras simbólicas
   * Exemplo: "Se status=Failed && retry>2 => Alerta"

3. **Visualização Interativa (Streamlit ou WebUI React)**

   * Filtros: por data, status, resultado
   * Gráficos: linha do tempo, histograma de status, mapa de falhas

4. **Logs e Análise de Qualidade**

   * Output de warnings
   * Cálculo de métricas agregadas (sucesso/falha, tempo entre missões)

---

## 3. Fases de Execução

### Fase 1: Parsing e Estruturação

* Implementar `log_parser.py`
* Criar testes unitários

### Fase 2: Enriquecimento VIREON

* Criar regras em `rules/mission_rules.json`
* Conectar com o `RuleManager`

### Fase 3: Visual Interface

* Criar dashboard interativo
* Exibir dados filtráveis e interativos

### Fase 4: Finalização

* Documentação (README)
* Deploy simples (Dockerfile ou HuggingFace Spaces)

---

## 4. Extras que Impressionam

* Adicionar modo CLI: `python mission_analyzer.py log.txt --summary`
* Exportação para CSV/JSON dos resultados filtrados
* Ranking das missões por complexidade/sucesso

## 5. Diferenciais do Candidato
